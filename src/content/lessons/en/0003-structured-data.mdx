<p class="recap">Recap from <a href="/lessons/0002-crawlable-vs-indexable.html">Lesson 0002</a>: your page is now crawlable and indexable. That gets it <em>into</em> the system. Structured data is the first lever that makes it stand out — and it's the first thing that serves <b>both</b> pipelines at once.</p>

<p class="lede">Structured data is machine-readable markup you embed in a page that says, explicitly, "this is a Product, its price is 29 USD, it's in stock." No NLP guesswork. Google recommends one format: <b>JSON-LD</b> — a `<script>` block of JSON, decoupled from your visible HTML.<sup>[1]</sup></p>

<Callout variant="win" lead="Your win:">generate valid JSON-LD for a page and lint it against Google's required-property rules — catching "this won't earn a rich result" <em>before</em> you ship, not weeks later in Search Console.</Callout>

## Prose vs. labelled fact

What a crawler sees in prose: *"Our Acme Widget Pro is available now for just twenty-nine dollars…"* — an engine must <b>infer</b>: is that a product? a price? which currency? Is it in stock? Maybe right, maybe not. The same fact as JSON-LD leaves nothing to guess:

```json
{
  "@type": "Product",
  "name": "Acme Widget Pro",
  "offers": {
    "price": "29.00",
    "priceCurrency": "USD",
    "availability": "InStock"
  }
}
```

<p class="fineprint">Reality check: Google says structured data is <em>not</em> a magic AEO trick and there's no special "AI markup" — clean structured data simply makes your existing content easier for any machine to use.<sup>[3]</sup> That's the honest version, and it's enough.</p>

## Required vs. recommended

Every type has properties Google marks <Pill kind="req" /> (miss one → no rich result at all) and <Pill kind="rec" /> (skip → still valid, just less rich). Knowing which is which is the whole game — and exactly what a linter automates.

| Type | What it needs |
|------|---------------|
| `FAQPage` | <Pill kind="req" /> `mainEntity` → each `Question` needs `name` + `acceptedAnswer.text` |
| `Product` | <Pill kind="req" /> `name`, and in `offers`: `price` + `priceCurrency` |
| `Article` | <Pill kind="rec" /> `headline`, `image`, `datePublished`, `author` |

## The skill: generate & lint with a script

In this workspace: `tools/schema_tool.py` — stdlib only. It emits a copy-paste sample for a type and lints any JSON-LD file against the rules above.

<Callout variant="do" lead="Do this now:">

1. Self-check: `python3 tools/schema_tool.py --demo`
2. Get markup you can paste into a page `<head>`: `python3 tools/schema_tool.py sample FAQPage`
3. Break it on purpose — save a Product JSON-LD with no `name`, then `python3 tools/schema_tool.py lint bad.json` → watch it flag the <b>ERROR</b>.
