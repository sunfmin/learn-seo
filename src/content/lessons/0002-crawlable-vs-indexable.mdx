<p class="recap">Recap from <a href="/lessons/0001-the-two-pipelines.html">Lesson 0001</a>: classic search runs <b>crawl → index → rank → serve</b>. Today we zoom into the first two — because a page that fails either can <em>never</em> rank, and as a builder these are the two gates your tooling can check automatically.</p>

<p class="lede">Developers conflate these constantly. "Crawlable" and "indexable" are <b>independent</b>. A page can be perfectly crawlable and still refuse to be indexed. A page can be marked index-me and never get crawled. They are two different gates with two different controls.<sup>[1]</sup></p>

<Callout variant="win" lead="Your win:">run a script that audits any URL through both gates and tells you exactly which one is failing — the tightest possible feedback loop on "can this page even show up?"</Callout>

<Callout variant="gotcha" lead="The trap that breaks everyone:">blocking a page in `robots.txt` does <u>not</u> remove it from the index. If the bot is blocked from crawling, it never <em>reads</em> your `noindex` tag — so to deindex a page you must <b>allow</b> the crawl and serve `noindex`. Block + noindex together = noindex never seen.<sup>[2]</sup> Crawl control and index control point in opposite directions.</Callout>

## The checklist, and what controls each

| Gate | Check | Control lives in |
|------|-------|------------------|
| <span class="badge c">CRAWL</span> | Path allowed | `/robots.txt` |
| <span class="badge c">CRAWL</span> | Returns 200 | HTTP status |
| <span class="badge i">INDEX</span> | No `noindex` | `<meta name="robots">` or `X-Robots-Tag` header |
| <span class="badge i">INDEX</span> | Has title | `<title>` in HTML |
| <span class="badge i">INDEX</span> | Has canonical | `<link rel="canonical">` |

## The skill: audit a URL with a script

You already have it — `tools/crawl_audit.py` in this workspace. Stdlib only, no install. The core: `urllib.robotparser` answers the crawl gate, a tiny HTML parser answers the index gate.

```python
# the heart of it — robots gate, then index signals
def robots_allows(url):
    rp = urllib.robotparser.RobotFileParser()
    rp.set_url(origin(url) + "/robots.txt")
    rp.read()
    return rp.can_fetch("Googlebot", url)   # stdlib does the parsing

# index gate: noindex? title? canonical?  (from the page's <head>)
```

<Callout variant="do" lead="Do this now —">
the feedback loop is the lesson:

1. Self-check the parser (offline): `python3 tools/crawl_audit.py --demo`
2. Audit a page that passes — Google's own docs: `python3 tools/crawl_audit.py https://developers.google.com/search/docs/fundamentals/how-search-works`
3. **Now audit a page you own.** Then deliberately break it: add `<meta name="robots" content="noindex">` to a test page and watch the INDEX gate flip to FAIL while CRAWL still passes. That contrast *is* the lesson.
</Callout>
