<p class="recap">回顾 <a href="/zh/lessons/0004-writing-for-retrieval.html">第 0004 课</a>：你让一个页面获得了入选资格（<a href="/zh/lessons/0002-crawlable-vs-indexable.html">0002</a>），给它的事实打上了标注（<a href="/zh/lessons/0003-structured-data.html">0003</a>），并把它的文字塑造成便于 retrieve（检索）的形态（0004）。这一切都是一场赌注。这一课要闭合这个循环——你怎么<em>知道</em>它真的落地了？</p>

<p class="lede">两条管线的衡量方式并不相同，而这道差距就是整个故事。对于经典搜索，Google 通过一个 API 给你 <b>ground truth（真实基准）</b>。对于 AI 答案的 citation（引用），<b>没有任何引擎对外提供这样的接口</b>——所以市面上每一个追踪工具，以及你即将搭建的那个，都是一个 <em>proxy（代理指标）</em>。<sup>[1]</sup></p>

<Callout variant="win" lead="你的收获：">在一组 answer-engine（答案引擎）的回复上运行 `citation_share.py`，得到你的 <em>coverage（覆盖率）</em>（引用了你的 prompt 占比）和 <em>share of voice（声量占比）</em>（你在所有引用中所占的份额）相对于竞争对手的数据——这正是 Profound 和 Ahrefs Brand Radar 在卖的那个 proxy 指标。</Callout>

## 经典这一侧：直接调用 API

这是个已经解决了的问题——先认证，然后 `POST` 一个日期区间和你想要的维度。没有工具需要搭建；平台本身就是工具。

```http
# Search Console: top queries by clicks, last 28 days
POST https://www.googleapis.com/webmasters/v3/sites/{siteUrl}/searchAnalytics/query
{
  "startDate": "2026-05-20", "endDate": "2026-06-17",
  "dimensions": ["query"],
  "rowLimit": 100
}
# -> rows of { keys:[query], clicks, impressions, ctr, position }
```

按计划定时拉取它，把这些行存起来，你就有了随时间变化的排名/曝光/点击趋势——任何 SEO 监控系统的骨干。

## AEO 这一侧：你要搭建的 proxy

因为没有 ground truth，每一个商用 AEO 追踪工具都做同样的三步——`tools/citation_share.py` 也是如此：

| 步骤 | 它是什么 | 谁负责 |
|------|-----------|-------------|
| 1 · prompt 库 | 你所在领域里一组固定的买家问题 | 你（API key + prompt） |
| 2 · 跑引擎 | 把每个 prompt 发出去，捕获回复 + 被引用的链接 | 你（一次 API 调用） |
| 3 · 解析并打分 | 提取被引用的域名 → coverage + share of voice | `citation_share.py` |

第二个辅助脚本 `tools/build_prompts.py` 负责第 1 步：它把那套买家问题 prompt 库组装出来——来自一次 LLM fan-out（扇出）或你的 Search Console 行数据——并且采用 `citation_share.py` 正好能消费的形态。这个工具就是那个离线、可测试的核心。它对 <em>被引用的链接</em>（AEO 里相当于排名的东西）打分，而不是对单纯的品牌提及打分：

```python
# coverage = prompts that cite me; SOV = my citations / all citations
cited = cited_domains(entry)          # links in response + citations[]
hit   = bool(cited & my_domains)
sov   = mine / total_citations
```

<Callout variant="do" lead="现在就做：">

1. 自检（离线）：`python3 tools/citation_share.py --demo`
2. 手工做一个很小的 `results.json`——3–4 个买家问题，把从 ChatGPT/Perplexity 拿到的真实答案 + 链接粘进去——然后给它打分：`python3 tools/citation_share.py results.json --domain yourdomain.com`
3. **下周再跑一次。** 唯一重要的那个数字是你<em>自己 coverage 的趋势</em>——而不是绝对值。

<Callout variant="gotcha" lead="proxy ≠ ground truth——诚实地读你自己的数字。">你的 prompt 库是<em>你</em>挑选的一个样本，不是引擎真实的 query（查询词）分布，而且答案是非确定性的。所以绝对的 share of voice 没什么意义，两个工具的数字也不可比——它们跑的是不同的 prompt。<sup>[1]</sup>把它当成一次民意调查来对待：一套固定的 prompt 集随时间反复跑，能给出一个可信的<em>趋势</em>；单次跑出来那个醒目的百分比则不能。把 prompt 保持不变，否则你衡量的是你的 prompt，而不是你的站点。</Callout>

<p class="fineprint">要知道的天花板：`citation_share.py` 只统计被引用的<em>链接</em>。“品牌被提及但没有链接”是另一种（更软的）指标；域名匹配很粗糙（没有 public-suffix 列表）。这两点都在工具的 docstring 里被标注为后续可升级的方向。</p>
