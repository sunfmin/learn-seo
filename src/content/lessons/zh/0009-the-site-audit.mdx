<p class="recap">回顾：第 <a href="/zh/lessons/0002-crawlable-vs-indexable.html">0002</a>–<a href="/zh/lessons/0008-eeat.html">0008</a> 课各自拿一个信号——robots/noindex、title、canonical、structured data（结构化数据）、一个 <code>&lt;h1&gt;</code>、正文篇幅——在<em>单个</em> URL 上检查。那是单元测试。这一课是集成测试。</p>

<p class="lede">真实站点有几百个页面，而出问题的那个信号，很少恰好坏在你随手看的那一页上——它坏在<b>模板（template）</b>里，所以它坏在<em>每一个</em>用了该模板的页面上。要发现这一点，你不能只审查一个 URL。你得像 bot 那样 crawl 整站，并<b>对每个页面把所有 gate 一次性跑完</b>。</p>

<Callout variant="win" lead="本课收获：">把一个脚本指向一个域名，拿回一份逐页清单<em>以及</em>一份全站汇总——“canonical 在 2/3 页面上缺失”——把一个模糊的担忧变成一张排好序的修复清单。</Callout>

## 像 bot 一样 crawl：从 seed 做 BFS

crawler 并没有背下你的 sitemap。它从一个 URL 出发，读取链接，把没见过的排进队列，然后重复——广度优先——并**留在本站内**。两条规则让它守规矩：

- **单一域名。** 只跟随 host 与 seed 一致的链接（用 `seolib.domain`）；跳过站外链接——别人的站点不归你审查。
- **设上限。** 跑满 N 个页面就停。礼貌的 crawl 是有界的；无界的 crawl 会把你本想帮的服务器锤爆。

```python
# the heart of it — BFS over internal links, capped
host = domain(seed)
seen, out, queue = set(), [], [seed]
while queue and len(out) < max_pages:
    url = queue.pop(0)
    if url in seen: continue
    seen.add(url)
    resp = fetch(url, ua="Googlebot")
    pg = parse(resp.body)                      # one seolib parse → every signal
    out.append((url, resp, pg))
    for href in pg.links:
        nxt = urljoin(url, href)               # relative → absolute
        if domain(nxt) == host and nxt not in seen:
            queue.append(nxt)                  # internal + unseen → enqueue
```

## 每个关卡，每个页面

对每个 crawl 到的页面，跑完整张清单——就是各课工具各自拥有的那些判定，如今汇成一张表：

| 关卡 | 检查 | 最早出现于 |
|------|------|-----------|
| <span class="badge c">CRAWL</span> | 页面返回成功（2xx/3xx） | `crawl_audit.py` · 0002 |
| <span class="badge i">INDEX</span> | 没有 `noindex`（meta 或 `X-Robots-Tag`） | 0002 |
| <span class="badge i">INDEX</span> | 有 `<title>` | 0002 |
| <span class="badge i">INDEX</span> | 声明了 `canonical` | 0002 |
| <span class="badge i">INDEX</span> | 恰好一个 `<h1>` | 结构 · 0004 |
| <span class="badge i">INDEX</span> | 有 JSON-LD structured data | `schema_tool.py` · 0003 |
| <span class="badge i">INDEX</span> | 正文篇幅够，能 retrieve（检索）出一个 chunk（块） | `geo_lint.py` · 0004 |

<Callout variant="insight" lead="汇总才是重点：">一个页面没过某项检查，是个笔误。<em>同一项</em>检查在许多页面上都失败，是<b>模板 bug</b>——而那是你手上杠杆率最高的修复，因为改对一个布局文件就能让所有页面一起痊愈。逐页清单找笔误；全站汇总找模板 bug。</Callout>
